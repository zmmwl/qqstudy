# 自然灾害防范主题 — AI互动科普平台 实施计划

## 一、项目概述

### 1.1 项目名称
**"安全守护者" — 自然灾害防范 AI 互动科普平台**

### 1.2 项目定位
面向各学段学生的纯软件/数字化自然灾害防范教育平台，集成 AI 文本生成、图像/视频生成、语音交互、数字孪生模拟等功能，以互动式、沉浸式体验传播防灾减灾知识。

### 1.3 核心价值
- **零门槛操作**：所有工具均为 Web 端操作，无需安装额外软件或硬件
- **AI 驱动**：利用 AI 能力自动生成科普内容，降低创作门槛
- **沉浸体验**：通过数字孪生场景模拟、语音交互等增强代入感
- **全面覆盖**：涵盖地震、洪水、台风、暴雨、雷电、火灾、泥石流、滑坡、高温热浪、寒潮等常见自然灾害

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────┐

│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐  │

│  │  工具模块 │ │ 生成模块  │ │  工具模块 │ │ 模拟模块│  │
│  └────┬─────┘ └────┬─────┘ └────┬─────┘ └───┬────┘  │
│       │            │            │            │       │
│  ┌────▼────────────▼────────────▼────────────▼────┐  │
│  │            AI 服务接口层 (API 聚合)              │  │
│  │   文本API │ 图像API │ 语音API │ 3D渲染引擎      │  │
│  └─────────────────────────────────────────────────┘  │
│  ┌─────────────────────────────────────────────────┐  │

```

### 2.2 技术选型

| 模块 | 技术方案 | 说明 |
|------|---------|------|
| 前端框架 | 原生 HTML5 + CSS3 + JavaScript | 零依赖，浏览器直接运行 |
| AI 文本生成 | OpenAI API / 本地提示词模板 | 支持在线API与离线模板双模式 |
| AI 图像生成 | Canvas API + SVG 动画 | 纯前端实现科普动画 |
| 视频生成 | Canvas录制 + MediaRecorder API | 前端实时录制科普动画视频 |
| 语音交互 | Web Speech API | 浏览器原生语音识别与合成 |
| 数字孪生 | Three.js (3D) / CSS 3D Transform | 轻量级 3D 场景模拟 |
| 数据存储 | localStorage / IndexedDB | 本地持久化用户数据 |
| 图标/样式 | 内联 SVG + CSS 动画 | 无外部依赖 |

## 三、功能模块设计

### 3.1 模块一：AI 文本生成工具

#### 功能列表
1. **防灾文案生成器**
   - 输入：灾害类型 + 目标受众（小学/初中/高中）
   - 输出：适配该学段的防灾宣传文案
   - 预置 10 种灾害类型的提示词模板

2. **应急预案方案生成器**
   - 输入：灾害类型 + 场景（学校/家庭/户外）
   - 输出：结构化的应急预案文档（Markdown格式）
   - 支持导出为文本文件

3. **安全知识问答库生成器**
   - 输入：灾害类型 + 题目数量 + 难度等级
   - 输出：选择题/判断题/简答题题库
   - 内置答题与评分功能

#### 页面设计
- 左侧：参数配置面板（灾害选择、受众选择、生成选项）
- 右侧：生成结果展示区（支持复制、导出）
- 底部：历史记录

---

### 3.2 模块二：图像/视频生成工具

#### 功能列表
1. **科普动画生成器**
   - 预置灾害科普动画模板（地震波传播、洪水上涨、台风形成等）
   - 支持自定义文字标注与配色
   - Canvas 实时渲染，可导出为 GIF/视频

2. **场景还原视频制作**
   - 提供灾害场景图示模板（横版/竖版）
   - 支持添加文字说明、时间线标注
   - 可叠加安全提示信息
   - 一键生成演示动画视频

3. **信息图表生成器**
   - 自动生成灾害数据统计图表
   - 支持自定义数据输入
   - 输出为高清图片

#### 页面设计
- 顶部：模板选择栏（灾害类型切换）
- 中部：画布预览区（实时预览）
- 右侧：属性编辑面板（颜色、文字、动画速度等）
- 底部：导出控制栏

---

### 3.3 模块三：语音交互工具

#### 功能列表
1. **语音防灾问答**
   - 语音输入问题 → 文本转换 → 匹配知识库 → 语音播报回答
   - 预置 200+ 条防灾知识条目
   - 支持多轮对话

2. **安全提醒播客生成器**
   - 选择灾害类型 → 自动生成播客文案 → TTS 语音合成
   - 支持选择不同声音风格（男声/女声/童声）
   - 可调节语速与语调
   - 支持导出音频文件

3. **声音复刻演示**
   - 录制短语音样本 → 基于模板生成个性化安全提醒
   - 预置多种提醒场景（地震来了、暴雨预警等）

#### 页面设计
- 主界面：语音对话气泡界面（类聊天应用）
- 侧栏：功能切换（问答/播客/声音复刻）
- 底部：语音控制栏（麦克风、播放、设置）

---

### 3.4 模块四：数字孪生/虚拟场景工具

#### 功能列表
1. **简易应急场景搭建**
   - 3D 场景编辑器：拖拽放置建筑、树木、道路等元素
   - 叠加灾害效果（地震裂缝、洪水水位、火焰粒子等）
   - 支持多视角切换

2. **逃生模拟演练**
   - 预置场景：学校教室、家庭住宅、户外山地等
   - 触发灾害事件 → 用户选择逃生路线
   - 实时评分：路线正确性、时间效率、安全规范
   - 结果反馈与知识点讲解

3. **灾害演变模拟**
   - 时间轴控制：展示灾害从发生到结束的全过程
   - 参数调节：调节灾害强度，观察不同影响
   - 对比展示：有防范 vs 无防范的差异

#### 页面设计
- 主区域：3D/2D 场景画布
- 左侧：场景元素工具栏
- 右侧：属性面板 + 知识提示卡
- 底部：时间轴 + 播放控制

## 六、实施步骤

### 阶段一：基础框架搭建（优先级：高）
1. 创建主入口页面 `index.html` — 统一导航门户
2. 搭建全局样式 `common.css` — 统一视觉风格
3. 编写公共工具函数 `common.js` — 路由、存储、通知等
4. 构建灾害知识数据库 `disaster-knowledge.json`

### 阶段二：文本生成模块（优先级：高）
1. 开发文案生成器 — 提示词模板 + 结果展示
2. 开发应急预案生成器 — 结构化文档输出
3. 开发问答库生成器 — 题目生成 + 答题交互
4. 实现导出功能（文本文件、Markdown）

### 阶段三：图像/视频生成模块（优先级：中）
1. 实现 Canvas 动画渲染引擎
2. 开发科普动画模板（先实现地震、洪水 2 个）
3. 开发场景还原视频模板
4. 实现视频录制与导出（MediaRecorder API）

### 阶段四：语音交互模块（优先级：中）
1. 集成 Web Speech API 语音识别
2. 开发语音问答功能（匹配知识库）
3. 开发播客生成器（TTS + 文案模板）
4. 实现音频导出功能

### 阶段五：数字孪生模块（优先级：中）
1. 实现 CSS 3D / Canvas 2D 场景渲染
2. 开发场景编辑器（拖拽放置元素）
3. 开发逃生模拟逻辑（路线选择 + 评分）
4. 实现灾害效果叠加（粒子动画）

### 阶段六：整合测试与优化（优先级：高）
1. 全模块联调测试
2. 响应式适配（桌面/平板/手机）
3. 性能优化（动画帧率、加载速度）
4. 用户体验打磨（提示文案、交互反馈）

## 七、设计规范

### 7.1 视觉风格
- **主色调**：安全蓝 `#2196F3` + 警示橙 `#FF9800`
- **辅助色**：成功绿 `#4CAF50` + 危险红 `#F44336`
- **背景**：浅灰 `#F5F5F5` + 白色卡片
- **字体**：系统默认无衬线字体栈
- **圆角**：统一 8px 圆角
- **阴影**：轻量级投影 `0 2px 8px rgba(0,0,0,0.1)`

### 7.2 交互规范
- 所有操作提供即时反馈（加载动画、成功/失败提示）
- 关键操作需二次确认（导出、重置等）
- 支持 Keyboard 快捷键（Enter提交、Esc关闭）
- 响应式布局，适配 1024px 以上屏幕

## 九、风险与应对

| 风险 | 应对策略 |
|------|---------|
| Web Speech API 兼容性 | 降级为文本输入模式，保留语音作为增强 |
| Canvas 动画性能 | 使用 requestAnimationFrame，限制粒子数量 |
| 知识库准确性 | 数据来源标注，声明仅供参考学习 |
| 文件体积过大 | 采用懒加载，按需加载模块资源 |
| AI API 不可用 | 提供离线模板模式，预置生成结果示例 |
